# CPP_Module_08 - Templated Containers

Это модуль про **STL (Standard Template Library)**
- стандартную библиотеку С++, к-рая предоставляет готовые инструменты для работы с данными.

В этом модуленужно обязательно использовать STL везде, где это уместно. Если писать свои алгоритмы поиска или сортировки - это будет плохо оценено.

Задача: научиться пользоваться существующими инструментами правильно.

# ex00 - Easy find

## Цель

 Написать ф-цию `easyfind`, к-рая ищет число в контейнере целых чисел. Разобраться с базвой работой STL.

## Что такое STL?

### Базовая идея

 Представим, что мы работаем с массивом чисел и нам часто нужно:
   - найти число в массиве
   - отсортировать массив
   - удалить элемент
   - скопировать часть массива

 Можно написать все функции самому, но это долго, скучно и чревато ошибками.

 **STL** - это библиотека, где уже всё написано. Мы просто используем готовый код, к-рый проверен и оптимизирован.

### Три кита STL

 STL состоит из 3 основных частей:

#### 1. **Containers (Контейнеры)**

 Контейнеры - это структуры данных, к-рые хранят элементы. Примеры:

 - **`std::vector`** - это как обычный массив, но динамический (растягивается если нужно)
   ```cpp
   		std::vector<int> v = {1, 2, 3, 4, 5};
   ```
 - **`std::list`** - это связный список (быстро удалять элементы из середины, но медленнее обращаться по индексу)
   ```cpp
   		std::list<int> lst = {1, 2, 3, 4, 5};
   ```
 - **`std::deque`** - это очередь, быстро добавлять в начало и в конец
 - **`std::set`** - множество уникальных элементов, отсортированных
 - **`std::map`** - словарь (ключ -> значение). В этом модуле их трогать не надо

 Общее свойство всех контейнеров: они шаблонные (работают с любым типом `T`).

#### 2. **Algorithms (Алгоритмы)**

 Алгоритмы - это готовые функции для работы с элементами контейнеров. Они определены в заголовке `<algorithm>`.
 Примеры алгоритмов:
  
 - **`std::find`** - найти элемент в контейнере
   ```cpp
		auto it = std::find(v.begin(), v.end(), 3);
		if (it != v.end())
			std::cout << "Found!" << std::endl;
   ```
 - **`std::sort`** - отсортировать контейнер
   ```cpp
   		std::sort(v.begin(), v.end());
   ```
 - **`std::count`** - подсчитать элементы
   ```cpp
   		int cnt = std::count(v.begin(), v.end(), 3);
   ```
 - **`std::reverse`** - развернуть контейнер
   ```cpp
   		std::reverse(v.begin(), v.end());
   ```

 Ключевой момент: все алгоритмы работают с *итераторами*.

#### 3. **Iterators (Итераторы)**

 Итератор - это что-то вроде указателя на элемент в контейнере. Можно думать о нём как о "объекте, к-рый указывает на элемент".

 Каждый контейнер предоставляет 2 метода:
 
 - **`begin()`** - итератор на первый элемент
 - **`end()`** - итератор на элемент *после* последнего (специальный маркер конца)

 ```cpp
	std::vector<int> v = {1, 2, 3};
	auto it = v.begin();	// указывает на 1
	++it;					// теперь указывает на 2
	*it;					// достаём элемент: 2

	auto end_it = v.end();	// указывает на "за границей" массива
	// end_it != v.begin() && end_it != v.end() - 1 (специальный маркер)
 ```

 **Зачем итераторы?**
 Алгоритмы написаны так, чтобы работать с любым контейнером. Вместо того, чтобы писать `std::find` для `vector`, отдельно для `list`, и т.д., пишут один алгоритм на итераторах. Это работает для всех контейнеров сразу!

 **Практический пример: `std::find`**
 ```cpp
	#include <vector>
	#include <algorithm>

	std::vector<int> numbers = {10, 20, 30, 40, 50};

	// Ищем число 30
	auto result = std::find(numbers.begin(), numbers.end(), 30);

	if (result != numbers.end())
		std::cout << "Found: " << *result << std::endl; // Выведет "Found: 30"
	else
		std::cout << "Not found" << std::endl;
 ```

 **Что произойдёт?**

 1. `std::find` начинает с `begin()`
 2. проходит по элементам: 10, 20, 30
 3. находит 30, возвращает итератор на этот элемент
 4. сравнивает с `end()` - это не конец, значит нашли
 5. дерефируем итератор (`*result`) и получаем значение

 Если число не найдено, `std::find` вернёт `end()`

## Что нужно сделать?

 Написать ф-цию `easyfind<T>`, к-рая:
   1. принимает контейнер `T` (например, `std::vector<int>` или `std::list<int>`)
   2. принимает целое число для поиска
   3. возвращает итератор на найденный элемент
   4. если элемент не найден, выбросить исключение (например, `std::exception`)

## Почему итератор?

 Итератор - это стандартный способ обозначить позицию в контейнере. Другие функции STL тоже возвращают итераторы. Это согласуется с философией STL.

## Почему исключение?

 Потому что это стандартный способ сообщить об ошибке в С++. Стандартные ф-ции (например, `std::vector::at`) тоже выбрасывают исключения при ошибках доступа.

 **Псевдокод**

 ```cpp
	function easyfind(container, value) {
		result = std::find(container.begin(), container.end(), value);
		if (result == container.end())
			throw std::exception();
		return result;
	}
 ```
 **Что это даёт?**

 После вызова `easyfind` получаем итератор, к-рый можем использовать дальше:
 ```cpp
	try {
		auto it = easyfind(myVector, 42);
		std::cout << "Found: " << *it << std::endl;
		// Можно использовать итератор в др. алгоритмах
		std::vector<int> newVec(it, myVector.end()); // всё от найденного до конца
	} catch (std::exception& e)
		std::cout << "Not found" << std::endl;

 ```

## Как это работает?

 1. Подключение заголовков: нужны `vector`, `list`, `algorithm`, `<exception>`.
 2. Использование `std::find`: это алгоритм из `algorithm`, принимает два итератора (начало и конец) и значение. Проходит по диапазону и возвращает итератор на первое совпадение или `end()`.
 3. Проверка результата: если итератор равен `end()`, значит не нашли.
 4. Выброс исключения: `throw std::exception();` прерывает ф-цию и передаёт управление в `catch` блок вызывающего кода.

## Дополнительные условия

 - ф-ция должна быть template, чтобы работать с любым контейнером целых чисел
 - не нужно обрабатывать ассоциативные контейнеры (`std::map`, `std::set`) - они работают по-другому
 - обязательно использовать `std::find` из `<algorithm>` - это ключевой момент модуля!
 - написать полноценные тесты, проверяющие разные сценарии

## Почему STL, а не свой код?

 - Надежность: STL писали эксперты, код проверен годами
 - Эффективность: алгоритмы оптимизированы, быстрее самоделок
 - Портативность: работает везде, где С++
 - Стандартизация: все программисты ожидают видеть STL, а не велосипеды

 В профессиональной разработке писать свой `find` - это анти-паттерн.


# ex01 - Span

## Цель

 Реализовать класс `Span`, к-рый хранит максимум `N` целых чисел и вычислять расстояния между ними:
   - `shortestSpan()` - минимальная разность (distance) между любыми двумя хранящимися числами в контейнере
   - `longestSpan()` - максимальная разность между любыми двумя числами в контейнере

 Конструктору передается единственный параметр `N` (unsigned int) - максимальную ёмкость контейнера. Метод `addNumber(int)` добавляет одно число; если контейнер уже заполнен, нужно бросить исключение. Если в контейнере меньше двух чисел, методы span должены бросать исключение.

## Что такое Span?

 `Span` - это ограниченный контейнер (container) для целых чисел.
 Идея - коробка фиксированного размера, в которую кладут числа не больше `N` значений, а потом вычисляют минимальное/максимальное расстояние между двумя числами внутри коробки. 
 
 Это не просто набор операций над элементами, а работа с коллекцией элементов.

## Что такое STL и почему оно здесь важно?
 
 Внутренне `Span` обычно реализуют через `std::vector<int>` и стандартные алгоритмы STL для эффективных вычислений.

 STL - это
   - `Containers` - контейнеры, к-рые хранят данные (`std::vector`, `std::list`, `std::map` и т.д.)
   - `Iterators` - итераторы, к-рые позволяют алгоритмам работать с любыми контейнерами
   - `Algorithms` - готовые алгоритмы для работы с контейнерами из `<algorithm>` (`std::sort`, `std::minmax_element`, `std::find` и т.д.)

 Здесь важно не писать всё вручную. Если мы делаем поиск, сортировку или собираем данные, то вместо своего цикла лучше использовать стандартный алгоритм. Это и есть главное требование модуля: применяем STL там, где это уместно.

## Что нового по сравнению с ex00?

 В ex00 мы писали простые шаблонные ф-ции (`swap`, `min`, `max`) - операции над отдельными значениями.

 Здесь мы реализуем полноценный класс-контейнер `Span` с управлением памяти и применением STL-algorithms для вычислений.
   - внутри него хранится набор значений
   - мы считаем не просто отдельные операции, а расстояния между элементами
   - теперь нужно правильно выбрать контейнер и алгоритмы, чтобы не писать собственную сортировку или собственный поиск min/max

 Это естественный прогресс: от элементарных шаблонов -> к реальному контейнеру с алгоритмической логикой и использованию `std::vector`, `std::sort`, `std::minmax_element`.

## Как это работает?

 1. `Span` создаётся с ёмкостью `N`.
 2. Метод `addNumber(int)` добавляет одно число.
 3. Если попытаться добавить больше `N` чисел - это ошибка.
 4. Для `longestSpan()` мы сравниваем самый маленький и самый большой элемент: максимальная разность всегда между крайними значениями.
 5. Для `shortestSpan()` мы ищем ближайшую пару чисел. Это значит, что удобнее сначала упорядочить числа, а затем посмотреть, какая пара соседних элементов даёт наименьшую разность.
 6. Если чисел меньше двух, то `span` найти нельзя, и это тоже ошибка.

 Важно: `Span` не должен хранить только одно число и не должен возвращать "что-то" вместо ошибки. Если вычисление `span` невозможно, нужно бросать исключение.

## Почему так?

 - потому что `Span` должен хранить наор, а не просто отдельное число
 - потому что `shortestSpan()` - это не просто поиск минимума, а сравнение расстояний между значениями
 - потому что модуль требует использовать STL-алгоритмы там, где это имеет смысл
 - потому что писать собственную сортировку и собственный поиск min/max в этом задании - плохой стиль

## Требования

- Класс `Span` реализован как реальный контейнер.
- Конструктор принимает только `unsigned int N`.
- Метод `addNumber(int)` добавляет один элемент.
- Если контейнер полон - `addNumber` выбрасывает исключение.
- Метод `size()` должен возвращать текущее кол-во элементов.
- `shortestSpan()` возвращает минимальную разность между двумя элементами.
- `longestSpan()` возвращает максимальную разность между двумя элементами.
- Если элементов меньше двух, оба метода бросают исключение.
- Тесты должны покрывать базовые случаи, переполнение, недостаточное количество данных и большой объём.

## Дполнительный функционал: диапазон итераторов

 Модуль STL показывает, что многие контейнеры (например, `std::vector`) поддерживают добавление целого диапазона элементов за раз через итераторы. Это удобнее, чем вызывать `addNumber()` тысячу раз.

 Надо реализовать в классе `Span` метод-template, к-рый принимает два итератора - начало и конец диапазона - и добавляет все элементы из этого диапазона в контейнер.

 Это типичный паттерн STL: контейнеры предоставляют ф-ции для работы с диапазонами.

 ```cpp
	// Таким образом, вместо:
	for (int i = 0; i < 1000; ++i)
		sp.addNumber(values[i]);

	// можно написать:
	std::vector<int> v = {1, 2, 3, 4, 5};
	sp.addRange(v.begin(), v.end());
 ```

 Если при добавлении диапазона контейнер переполняется, выброси исключение.

## Почему это хорошая практика?

 - `Span` показывает, как работать с коллекцией данных, а не с одиночными значениями
 - это упражнение учит применять `std::vector` и алгоритмы STL, а не "изобретать велосипед"
 - в профессиональном коде именно такой подход ценится больше всего: использовать стандартные инструменты, а не писать всё вручную.

# ex02 - ...

## Цель

 Реализовать `MutantStack` - класс, к-рый ведет себя как `std::stack`, но при этом даёт доступ к итераторам.

 Иначе говоря, это обычный стек, к-рый можно обходить как контейнер STL.

## Что такое `std::stack`?

 `std::stack` - это адаптер контейнера, а не полноценный контейнер сам по себе. Он предоставляет только стековый интерфейс:

 - `push()`
 - `pop()`
 - `top()`
 - `size()`
 - `empty()`

 Главная идея стека - доступ только к верхнему элементу по принципу LIFO:
   - `Last In` - последним положили
   - `First Out` - первым забрали

 То есть стек не предназначен для обхода всех элементов через итераторы.

## В чём проблема?

 У `std::stack` нет `begin()` и `end()`, поэтому по нему нельзя пройтись `for`, нельзя использовать стандартные алгоритмы STL и нельзя легко вывести все элементы.

 Это ограничение удобно для чистого стека, но неудобно, если нужно посмотреть содержимое целиком.

## Что нужно сделать?

 Нужно создать `MutantStack`, к-рый
   - работает как обычный `std::stack`
   - поддерживает все стандартные методы стека
   - дополнительно предоставляет итераторы

 То есть мы хотим получить стек, по к-рому можно пройтись от начала до конца.

## Как это обычно работает?

 `std::stack` - это адаптер, к-рый внутри хранит другой контейнер.
 По умолчанию это `std::deque`, но можно использовать и другой контейнер, если он поддерживает нужные операции.

 Ынутренний контейнер у `std::stack` доступен как защищённый член `с`, поэтому класс-наследник может получить к нему доступ и вернуть его итераторы:

 - `begin()` -> `this->c.begin()`
 - `end()` -> `this->c.end()`

 Таким образом, `MutantStack` становится iterable.

## Почему наследование?

 Потому что `std::stack` уже содержит всю нужную логику:

 - добавление эл-тов
 - удаление эл-тов
 - доступ к верхнему эл-ту
 - размер
 - проверка на пустоту

 Наследование позволяет не переписывать всё заново, а просто добавить недостающую возможность - итераторы.

## Какие итераторы нужны?

 Обычно добавляют:
   - `iterator`
   - `const_iterator`
   - `reverse_iterator`
   - `const_reverse_iterator`

 И соответствуюшие методы:
   - `begin()`
   - `end()`
   - `rbegin()`
   - `rend()`
   - и const-версии если нужно

## Почему это полезно?

 `MutantStack` показывает, как можно расширить поведение стандартного адаптера контейнера, не ломая его основную идею.

 Это задание учит:
   - работать с наследованием
   - понимать адаптеры контейнеров STL
   - использовать адаптеры
   получать доступ к внутреннему контейнеру через `c`

## Дополнительные условия
